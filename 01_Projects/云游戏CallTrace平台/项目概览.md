# 云游戏CallTrace平台

## 背景

云游戏 CallTrace 平台是当前研究的 Trace 方向项目，用于云游戏场景下的调用链追踪和问题定位。

该平台借鉴 5GC 通讯行业中的 CallTrace 技术，把一次云游戏串流过程建模为端到端 Trace，用于解决一次串流过程中的故障排查链路问题。

## 问题

需要沉淀 Trace 平台的项目上下文，帮助后续 Agent 理解其在云游戏链路中的定位、数据模型和排障价值。

## 方案

将云游戏 CallTrace 平台作为可观测性和排障类项目维护，后续重点沉淀：

- Trace 数据模型
- 调用链采集点
- 链路关联规则
- 查询和聚合能力
- 延迟、错误和异常路径分析
- 与云游戏实例、Agent、任务系统的关联

## 为什么这么做

Trace 平台的价值在于把分散的调用、实例状态和异常路径串起来。提前记录项目定位，有助于后续排障时建立统一的分析入口。

## 踩坑

暂无。

## 下次怎么复用

当讨论云游戏 CallTrace 平台时，Agent 应优先关注：

- Trace ID 如何生成、传递和关联
- 哪些关键调用必须打点
- Trace 数据如何与实例、任务、Agent 状态关联
- 查询结果是否能直接支持排障决策
- 采样、存储成本和查询性能之间的取舍

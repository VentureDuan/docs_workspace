# 01 | Redis 初识

## 一、Redis的特性

1. 速度快
2. 持久化
3. 多种数据结构
4. 支持多种编程语言(客户端)
5. 功能丰富
6. 简单
7. 主从复制
8. 高可用、分布式



### 1. 速度快

官方数据是 10w OPS。为什么它会这么快？

1. 数据存储：数据存储在内存中，内存的数据读写速度在 ns 级别，这是它速度快的本质原因。
2. 语言实现：使用 C 语言实现，代码质量高，性能强悍。
3. 线程模型：单线程实现，CPU不是当时Redis的性能瓶颈，并且多线程会产生线程切换、锁竞争等问题。



![image-20240423193130879](./static/image-20240423193130879.png)



### 2. 持久化

前面我们知道 Redis 数据存储在内存中，内存具有易失性，断电后数据会丢失。所以Redis提供持久化功能

Redis所有数据保持在内存中，对数据的更新将异步地保存到磁盘上。



### 3. 多种数据结构

Redis是键值型数据库，key只能是strings，但是value可以是多种数据结构(与它对应的是 memcached 只提供字符串类型的 value)：

<img src="./static/image-20240423194006210.png" alt="image-20240423194006210" style="zoom:50%;" />

除了上述5种数据结构，Redis在新版本还提供了其衍生的数据结构：

1. BitMaps: 位图 (布隆过滤器)
2. HyperLogLog：超小内存唯一值计数 (允许误差)
3. GEO：地理信息定位



### 4. 支持多种客户端语言

主流编程语言都支持



### 5. 功能丰富

- 发布订阅：实现很多消息的功能
- Lua脚本：自定义命令(Redis单线程执行)
- 事务
- pipeline：提高客户端并发效率



### 6. 简单

- 核心模块：2.3w行代码(去除集群功能)
- 不依赖外部库
- 单线程

### 7. 主从复制

### 8. 高可用、分布式

高可用：Redis从2.8版本后提供 Redis-Sentinel功能实现其高可用

分布式：Redis从3.0版本后支持 Redis-Cluster功能支持分布式



## 二、Redis安装和部署

### 0. 安装前的准备工作

```bash
# 安装 gcc
yum install gcc gcc-c++
```

### 1. Redis安装

```bash
# 从 Redis 下载站点获取最新稳定版本 Redis 的源文件
wget https://download.redis.io/redis-stable.tar.gz

# 编译 Redis，首先解压，切换到redis-stable，然后运行 make 
tar xf redis-stable.tar.gz
cd redis-stable
make

# 安装redis，其实就是将 src 目录下的二进制文件移到 /usr/local/bin 目录下面
sudo make install
```

### 2. 可执行文件说明

如果编译成功，你将在 src 目录下找到几个 redis 二进制文件，包括：

- redis-server: Redis服务器
- redis-cli：Redis命令行客户端
- redis-benchmark：性能测试功能
- redis-check-aof：AOF修复工具
- redis-check-rdb：RDB修复工具
- redis-sentinel：setinel服务器





### 3. 启动

一般我们推荐配置文件启动。将 redis.conf 移到 /etc/目录下。修改必要的配置

- 将 daemonize 属性值由 no 改为 yes，使 Redis 进程以守护进程方式运行。
- port：Redis对外端口号
- logfile：日志输出文件名
- dir：Redis工作目录
- bind，绑定客户端 IP，用于限定可以访问自己的客户端 IP。
- requirepass：设置访问密码



可以使用 `config get *` 查看Redis所有配置

给redis-server配置systemd管理

```bash
[root@localhost ~]# cat /usr/lib/systemd/system/redis.service
[Unit]
Description=Redis persistent key-value database
After=network.target

[Service]
ExecStart=/usr/local/bin/redis-server /etc/redis.conf --daemonize no
ExecStop=/usr/local/bin/redis-cli shutdown
User=redis
Group=redis
RuntimeDirectory=redis
RuntimeDirectoryMode=0755

[Install]
WantedBy=multi-user.target
```

expire、ttl、persist

```bash
# expire key seconds 给key设置在seconds秒后过期

10.0.0.12:db0> get name
today
10.0.0.12:db0> expire name 3
1
# 三秒后
10.0.0.12:db0> get name

# ttl key 查看key剩余过期时间
10.0.0.12:db0> get name
today
10.0.0.12:db0> expire name 60
1
# 还剩 56 秒过期
10.0.0.12:db0> ttl name
56
# -2 表示该key已经被删除了
10.0.0.12:db0> ttl name
-2

# persist key 去除key的过期时间
10.0.0.12:db0> persist name
1
# -1表示永不过期
10.0.0.12:db0> ttl name
-1
```

### 5. 列表类型

**列表键值类型**

key-value对，value是一个双端队列，可以从左边插入和删除，也可以从右边插入和删除



**特点：**

- 有序
- 可以重复
- 可以从左右弹出



**rpush**

```bash
# rpush key value1 value2 .. valueN
# 从列表右端依次插入(1~N个)
127.0.0.1:6379> rpush listkey c b a
(integer) 3
127.0.0.1:6379> lrange listkey 0 2
1) "c"
2) "b"
3) "a"
```

**lpush**

```bash
# lpush key value1 value2 .. valueN
# 从列表左端依次插入(1~N个)
127.0.0.1:6379> lpush numkey 1 2 3
(integer) 3
127.0.0.1:6379> lrange numkey 0 2
1) "3"
2) "2"
3) "1"
```

 **linsert**

```bash
# linsert key befor|after value newValue
# 在list指定的值 前|后 插入 newValue
127.0.0.1:6379> linsert listkey before b 10
(integer) 4
```

**lpop**

```bash
# lpop key
# 从列表左侧弹出一个item
127.0.0.1:6379> lpop listkey
"c"
```

rpop

```bash
# lpop key
# 从列表右侧弹出一个item
127.0.0.1:6379> rpop listkey
"a"
```

**lrem**

```bash
# lrem key count value
# 根据count值，从列表中删除所有value相等的项
1. count > 0, 从左到右，删除最多count个value相等的项
2. count < 0, 从右到左，删除最多Math.abs(count)个value相等的项
3. count = 0, 删除所有value相等的项

127.0.0.1:6379> lrem listkey -1 e
(integer) 1
```

**ltrim**

```bash

```

**TIPS:**

- lpush + lpop = stack
- lpush + rpop = queue
- lpush + ltrim = Capped Collection(固定数量列表)
- lpush + brpop = Message Queue

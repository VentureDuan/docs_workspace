# CGInstanceAgent 实例Agent服务

## 背景

CGInstanceAgent 是当前研究的实例 Agent 服务，负责实例自检、状态上报、自更新和 Temporal 任务流程编排。

Agent 核心是一个循环：感知环境、做决策、执行动作、观察结果、再次决策。

## 问题

需要沉淀实例侧 Agent 的长期上下文，尤其是 Agent 循环、状态上报、自更新和 Temporal 编排之间的关系。

## 方案

将 CGInstanceAgent 作为实例侧自治 Agent 项目维护，后续围绕以下主题追加：

- Agent 主循环
- 环境感知和实例自检
- 决策策略
- 动作执行和结果观察
- 状态上报协议
- 自更新流程
- Temporal 工作流和活动边界

## 为什么这么做

该项目的核心不是单次任务执行，而是持续闭环。Agent 的设计需要同时考虑可靠性、可恢复性、幂等性、可观测性和升级安全。

## 踩坑

暂无。

## 下次怎么复用

当讨论 CGInstanceAgent 时，Agent 应优先使用以下思维框架：

```text
感知环境 -> 做决策 -> 执行动作 -> 观察结果 -> 再次决策
```

评估代码或方案时重点关注：

- 每一步是否可观测
- 决策是否可解释
- 动作是否幂等
- 失败是否能重试或补偿
- 自更新是否能回滚
- Temporal 编排是否避免把不稳定逻辑放进工作流确定性路径
